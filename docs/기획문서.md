# 마비노기 모바일 아이템 데이터베이스 웹 페이지 기획 문서

## 1. 개요
본 문서는 마비노기 모바일 아이템 데이터베이스를 기반으로 사용자가 현재 보유한 아이템을 활용하여 제작 가능한 상위 아이템 정보와, 특정 아이템 제작에 필요한 하위 아이템 정보를 시각적으로 제공하는 웹 페이지 개발을 위한 기획을 담고 있습니다.

## 2. 목표
- 사용자가 보유한 아이템 목록 기반으로 제작 가능한 상위 아이템 시각화
- 특정 상위 아이템 선택 시, 필요 하위 아이템 목록 및 수량 정보 제공
- 아이템 상세 정보 (속성, 효과 등) 조회 기능 제공
- 직관적이고 사용자 친화적인 UI/UX 제공

## 3. 핵심 기능

### 3.1. 내 아이템 관리
- 사용자가 보유한 아이템을 추가/삭제/수정할 수 있는 기능
- 아이템 ID, 수량 등을 입력하여 관리

### 3.2. 제작 가능 아이템 조회
- 내 아이템 목록을 기반으로 제작 가능한 모든 상위 아이템 목록을 표시
- 제작 가능한 아이템은 조합 성공률, 필요 시설 레벨 등 추가 정보와 함께 표시

### 3.3. 아이템 제작 트리 조회
- 특정 제작 가능 아이템 선택 시, 해당 아이템 제작에 필요한 모든 하위 아이템 목록 (재료, 수량)을 트리 형태로 시각화
- 부족한 재료는 별도 표시

### 3.4. 아이템 상세 정보 조회
- 아이템 클릭 시, 해당 아이템의 상세 정보 (공격력, 방어력, 특수 옵션, 효과 등) 팝업 또는 별도 페이지로 표시
- 강화/인챈트 정보, 세트 효과 정보 등 포함

### 3.5. 시장 데이터 연동 (향후 확장)
- 아이템별 평균 거래 가격, 거래량 등 시장 데이터 표시 (옵션)

## 4. 데이터베이스 연동
기존에 제공된 마비노기 모바일 아이템 데이터베이스 스키마를 기반으로 백엔드 시스템을 구축하고 웹 페이지와 연동합니다.

## 5. UI/UX 설계 (와이어프레임 아이디어)

### 5.1. 메인 페이지
- 좌측: 내 아이템 목록 관리 영역 (아이템 추가/삭제 버튼, 목록)
- 우측: 제작 가능 아이템 목록 (상위 아이템 아이콘, 이름, 성공률, 시설 요구조건 등)

### 5.2. 아이템 제작 트리 팝업/페이지
- 선택된 상위 아이템 정보 표시
- 트리 형태의 재료 목록: 각 재료 아이템 이름, 필요 수량, 보유 수량 (색상으로 부족 여부 표시)

### 5.3. 아이템 상세 정보 팝업/페이지
- 아이템 아이콘, 이름, 등급
- 속성 (공격력, 방어력, 크리율, 원소 저항 등)
- 특수 옵션, 세트 효과
- 강화/인챈트 현황

## 6. 기술 스택 (예시)
- **프론트엔드**: React, Next.js, Tailwind CSS (또는 Ant Design, Material-UI 등)
- **백엔드**: Node.js (Express), Python (Django/Flask), Java (Spring Boot) 등
- **데이터베이스**: MySQL, PostgreSQL (기존 스키마 활용)

## 7. 개발 일정 (예상)
- 1주차: 데이터베이스 연동 및 기본 API 개발
- 2주차: 프론트엔드 기본 구조 및 UI 구현 (내 아이템 관리, 제작 가능 아이템 목록)
- 3주차: 아이템 제작 트리, 상세 정보 조회 기능 구현
- 4주차: 테스트 및 배포

## 8. 테스트 계획
- 각 API 엔드포인트별 유닛 테스트
- 프론트엔드 컴포넌트별 통합 테스트
- 사용자 시나리오 기반 E2E 테스트

## 9. 향후 확장 고려사항
- 아이템 검색 기능
- 즐겨찾기 아이템 목록
- 사용자 로그인 및 데이터 저장 기능
- 시장 데이터 실시간 연동
